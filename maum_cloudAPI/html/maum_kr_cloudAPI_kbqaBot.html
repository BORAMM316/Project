API 공통 가이드

개발 환경 세팅

1) REST API는 HTTP 요청을 보낼 수 있는 환경이라면 어디에서든 이용할 수 있습니다.

2) 다양한 환경의 웹 서버에서 활용 가능합니다. (Python, Web, Javascript, Java)

키 발급

1) Mindslab에서 제공되는 API 서비스를 사용하기 위해서는 키를 선지급 받아야합니다.

2) 마음AI 플랫폼에서 Business Plan 이상 구독 신청을 하셔야합니다. (https://maum.ai)

3) [계정메뉴]에서 본인의 API ID 및 Key를 확인해주세요.

4) 발급받은 API ID 및 Key를 기억하여 아래 매뉴얼에 맞게 사용합니다.

※ Mindslab API를 위한 ID와 키는 외부 유출 및 공유를 금합니다.

KBQA (Knowledge Base Question Answering)

위키피디아의 인포박스를 KBQA 엔진을 이용하여 검색하여 답변해줍니다.

준비사항

- Input: 텍스트(질문)

- 아래 Model 중 택 1

  • 위키/백과 (default)
실행 가이드

① Request

  • Method : POST
  • URL : https://api.maum.ai/kbqa/answer Request

② Request 파라미터 설명

키 설명 type
apiId 사용자의 고유 ID. 마인즈랩 담당자에게 이메일로 발급요청 string
apiKey 사용자의 고유 key. 마인즈랩 담당자에게 이메일로 발급요청 string
lang language (eng | kor) 현재는 kor만 지원 string
question 질문 string

③ Request 예제

curl -X POST \
https://api.maum.ai/api/irqa/ \
-H 'content-type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW' \
-F ID=(*ID 요청 필요) \
-F key=(*key 요청 필요) \
-F lang=kor \
-F question=사자왕의 출생지는? \

④ Response 파라미터 설명

Response
키 설명 type
message API 동작여부 string
payload 결과값 string
message: API 동작여부
키 설명 type
message 요청 처리 상태를 설명하는 문자열 (Success/ Fail) string
status 요청 처리 상태에 대한 status code (0: Success) long
payload: 결과
키 설명 type
payload 결과값 string

⑤ Response 예제

{
    "message": {
        "message": "Success",
        "status": 0
    },
    "payload": "영국 옥스퍼드"
}
