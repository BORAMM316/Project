3D 인테리어

3D 인테리어 (Holistic 3D)

방 안을 360도에서 촬영한 파노라마 이미지를 3D 모델로 생성합니다.

샘플 파일로 테스트 하기

이미지 출처: https://pixexid.com

내 파일로 해보기

  • * 지원가능 파일 확장자: .png
  • * 이미지 용량 2MB 이하만 가능합니다.

이미지 분석중

약간의 시간이 소요 됩니다. (약 3초 내외)

결과 파일

결과 파일 다운로드
* 결과 파일은 .pcd 파일로 지원합니다. (.pcd 파일은 https://www.creators3d.com/online-viewer 에서 파일을 업로드하여 확인 하실 수 있습니다.)
API 공통 가이드

개발 환경 세팅

1) REST API는 HTTP 요청을 보낼 수 있는 환경이라면 어디에서든 이용할 수 있습니다.

2) 다양한 환경의 웹 서버에서 활용 가능합니다. (Python, Web, Javascript, Java)

키 발급

1) Mindslab에서 제공되는 API 서비스를 사용하기 위해서는 키를 선지급 받아야합니다.

2) 마음AI 플랫폼에서 Business Plan 이상 구독 신청을 하셔야합니다. (https://maum.ai)

3) [계정메뉴]에서 본인의 API ID 및 Key를 확인해주세요.

4) 발급받은 API ID 및 Key를 기억하여 아래 매뉴얼에 맞게 사용합니다.

※ Mindslab API를 위한 ID와 키는 외부 유출 및 공유를 금합니다.

3D 인테리어 (Holistic 3D)

방 안을 360도에서 촬영한 파노라마 이미지를 3D 모델로 생성합니다.

준비사항

- Input: 이미지 파일

  • 확장자: .png
  • 용량: 2MB 이하
실행 가이드 Upload : pcd 3d model 생성 요청을 통해 생성 요청을 만들고, 처리 상태를 알기 위한 request key를 받습니다.

① Request

  • Method : POST
  • URL : https://api.maum.ai/holistic3d/upload

② Request 파라미터 설명

키 설명 type
apiId 사용자의 고유 ID. 마인즈랩 담당자에게 이메일로 발급요청 string
apiKey 사용자의 고유 key. 마인즈랩 담당자에게 이메일로 발급요청 string
image type:file (.png) 이미지 파일 file

③ Request 예제

curl --location --request POST 'https://api.maum.ai/holistic3d/upload' \
--header 'Content-Type: multipart/form-data' \
--form 'apiId= 발급받은 API ID' \
--form 'apiKey= 발급받은 API KEY' \
--form 'image= 3D를 인식할 이미지 파일' \

④ Response 파라미터 설명

Response
키 설명 type
message API 동작여부 object
payload 결과 object
message: API 동작여부
키 설명 type
message 요청 처리 상태를 설명하는 문자열 (Success/ Fail) string
status 요청 처리 상태에 대한 status code (0: Success) int
payload: 결과
키 설명 type
status follow status code of mmsr
status(1): message -> file_key
status(2): upload error 		int
fileKey 요청 식별용 고유 Key string
msg 요청에 대한 msg string

⑤ Response 예제

{
	"message": {
		"message": "Success",
		"status": 0
	},
	"payload": {
		"status": "1"
		"fileKey": "369f5394-dca7-4356-813e-60ae9a74be80"
		"msg": "success"
	}
}
Check process : 요청의 처리 상태를 확인합니다.

① Request

  • Method : POST
  • URL : https://api.maum.ai/holistic3d/check:process

② Request 파라미터 설명

키 설명 type
apiId 사용자의 고유 ID. 마인즈랩 담당자에게 이메일로 발급요청 string
apiKey 사용자의 고유 key. 마인즈랩 담당자에게 이메일로 발급요청 string
fileKey 요청시 발급받은 요청 식별용 key string

③ Request 예제

curl --location --request POST 'https://api.maum.ai/holistic3d/check:process' \
--header 'Content-Type: application/json' \
--form 'apiId= 발급받은 API ID' \
--form 'apiKey= 발급받은 API KEY' \
--form 'fileKey= 요청시 발급받은 요청 식별용 key' \

④ Response 파라미터 설명

Response
키 설명 type
message API 동작여부 object
payload 결과 object
message: API 동작여부
키 설명 type
message 요청 처리 상태를 설명하는 문자열 (Success/ Fail) string
status 요청 처리 상태에 대한 status code (0: Success) int
payload: 결과
키 설명 type
status NOT_YET = 6;
PROCESSING = 7;
DONE = 8;
WRONG_KEY = 10;
ERROR = 11;
DELETED = 9; 		int
message 상태 코드에 따른 문구 string

⑤ Response 예제

{
	"message": {
		"message": "Success",
		"status": 0
	},
	"payload": {
		"status": 8,
		"message": "Process Completed(can download)"
	}
}
Download: 처리 완료된 3D .pcd 파일을 다운로드 받습니다.

① Request

  • Method : POST
  • URL : https://api.maum.ai/holistic3d/download

② Request 파라미터 설명

키 설명 type
fileKey 요청시 발급받은 요청 식별용 key string

③ Request 예제

curl --location --request GET
'https://api.maum.ai/holistic3d/download?fileKey= 요청시 발급받은 요청 식별용 key'

④ Response 예제

Content-Type: application/octet-stream
(.pcd File Download)
